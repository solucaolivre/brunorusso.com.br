Abaixo estão algumas informações importantes que as pessoas que estão começando a utilizar GNU/Linux devem saber.

  1. A partição / (root), pode ser pequena. Não precisa de muito espaço, pois ela deve conter o seguinte:
    • /bin, contêm alguns programas (ls, free, tar etc) usados por todos os usuários.
    • /sbin, contêm arquivos requiridos pelo Sistema Operacional. Alguns são usados na hora do boot e para reparar o sistema.
    • /etc, contêm arquivos de configuração tanto do boot do sistema operacional quanto de outros programas.
    • /lib, contêm as bibliotecas utilizadas por diversos programas. Portanto, um bom espaço para essa partição seria de 500Mb à 1Gb. Como hoje em dia disco é razoavelmente barato, podemos deixar 1Gb.
  2. A partição /boot para um servidor e para uma estação de trabalho contêm arquivos estáticos e relativamente pequenos. Um tamanho ideal para essa partição seria de 100Mb.
  3. A partição /home vai variar conforme o seu uso. Por exemplo: para um servidor de arquivos essa partição pode ser relativamente grande, poisos usuários irão armazenar arquivos nela. Mas quando se faz a instalçao de um firewall, onde praticamente não haverá mais que 1 ou 2 usuários essa partição pode estar dentro da partição / (root).
    • Caso ela seja usada como no primeiro exemplo, é aconselhável que seja utilizado raid.
  4. A partição /tmp é utilizada por alguns programas para armazenar arquivos temporários. Em algumas distribuições como por exemplo a Debian, essa partição é limpa a cada boot no equipamento. Seu tamanho também pode variar dependendo do tipo de aplicação que for utilizado na máquina.
  5. A partição /var, na minha opinião ela é muito importante, pois é nesta partição que são armazenados os arquivos de LOG. Em um servidor o uso dessa partição é fundamental. Quase todos os serviços armazenam log nessa partição. Em caso de uma auditoria os arquivos que essa partição contêm são observados com muito cuidado.
  6. A partição /usr é a maior hierarquia nas partições que existe em um sistema Linux. Mas calma! Não é por isso que todo o espaço deve ser destinado à ela. Essa partição contêm grande parte dos programas e comandos utilizados pelos usuários, código fonte, exemplo de configuração e documentação sobre os programas instalados. Em ambientes corporativos essa partição pode ser compartilhada por diversos usuários. em alguns casos, administradores de sistema costumam deixar essa partição somente como leitura e ezecução. Meio radical, mas em alguns casos é necessário para evetiar que ocorra danificação nos dados.
